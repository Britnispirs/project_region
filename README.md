# Разработка программного комплекса для автоматизированного аудита безопасности веб-ресурсов
❗Для проверки кода понадобятся библиотеки bs4 и requests❗


Моя программа написана на языке Python и работает в несколько простых этапов. Сначала она подключается к сайту через библиотеку requests. Чтобы сайт не догадался, что его проверяет автоматический скрипт, я настроил программу так, чтобы она представлялась обычным браузером. Также я добавил в код команду, которая заставляет программу игнорировать ошибки безопасности сертификатов. Это важно, потому что на многих учебных сайтах защиты нет, и обычные программы на них просто вылетают, а мой сканер продолжает работу.

Затем программа начинает «читать» страницу с помощью библиотеки BeautifulSoup. Она ищет на сайте все места, куда пользователь может ввести текст — это обычные поисковые строки или формы входа. Скрипт сам находит названия полей и понимает, как именно нужно отправить данные серверу. Это очень удобно, потому что мне не нужно вручную прописывать детали для каждого нового сайта — программа делает это сама.

Когда список полей готов, начинается проверка. Программа по очереди берет разные вредоносные символы и впечатывает их в найденные формы. После каждой попытки сканер внимательно смотрит на то, что сайт прислал в ответ. Если в ответе от сервера появляются сообщения о сбоях в базе данных или если сайт возвращает обратно тот код, который я в него отправил, программа понимает: защита не сработала.

В самом конце программа выводит результат в консоль. Если она нашла проблему, то пишет, какая это уязвимость и где именно она находится. Я также добавил в код защиту от ошибок: если интернет пропадет или сайт перестанет отвечать, программа не закроется, а просто сообщит об этом и пойдет дальше. В итоге получился простой, но надежный инструмент, который делает всю рутинную работу по проверке безопасности за человека.
